## 0. 三条交叉验证结论(已坐实)
- `virtual_rs_window` 是真·原生窗口抽象,**iOS(.mm)+ Android(jni+java)双证**,非从零。
- **渲染层无需写 Metal**:`graphic_2d/.../platform/darwin/rs_surface_darwin.cpp`(iOS/macOS 共用)用 GLFW+skia GL **离屏渲染 → ReadPixels 回 CPU RGBA → `onRender_` 回调**。macOS 复用同一后端,只需 NSView 显示 buffer。
- M1 **可做独立可执行**(`main.mm` + MacAppDelegate),`build/toolchain/mac` 有 `tool("link")` 带 frameworks;**免 Xcode .app/.xcodeproj/签名**。

## 1. iOS 启动链(macOS 照此镜像)
```
main.m (UIApplicationMain) → AppDelegate didFinishLaunching
 → [StageApplication configModuleWithBundleDirectory:@"arkui-x"]  // 挂 .abc
 → [StageApplication launchApplication]                            // 起 AbilityRuntime
 → rootVC = StageViewController
     viewDidLoad:
       → StageContainerView(UIView 容器)
       → WindowView(UIView,真正渲染视图)+ startBaseDisplayLink(CADisplayLink 驱 vsync)
       → [windowView createSurfaceNode] → Window::CreateSurfaceNode(self.layer)
            → RSUIDirector::Create() + RSSurfaceNode::Create({additionalData=layer})
       → AppMain::DispatchOnCreate / DispatchOnForeground   // 跑 .ets 页面
```
事件:WindowView `touchesBegan/...` → dispatchTouches → AcePointerDataPacket → `Window::ProcessPointerEvent`。

## 6. 风险排序(高→低)
1. **【最高·先验证】graphic_2d 的 `platform/darwin` RS 后端能否 `target_os=mac` 编译运行** —— GLFW+skia GL 在 mac 可用,但 `USE_GLFW_WINDOW`/Vulkan 开关组合 + onRender 回调闭环(GPU→CPU buffer→贴回 layer)未完整追到。**Phase 0 第一刀:单独验证 graphic_2d darwin 目标能否为 mac 编过。**
2. **CADisplayLink→CVDisplayLink**(vsync 节拍,线程模型不同,必重写)。
3. **NSView backing layer**(`wantsLayer`+`makeBackingLayer`+`viewDidChangeBackingProperties`,与 iOS layoutSubviews 时机不同)。
4. 45 处 IOS_PLATFORM 取舍(复用+少量岔开)。
5. 坐标系 Y 翻转(`isFlipped` 漏设→点错位)。
6. toolchain sysroot(有 mac 先例,低)。
7. ace_tools 不支持 mac 打包(M1/M2 手写 main.mm + gn target 绕过)。

### GL vs Metal 决策(下一步的关键岔口)
- **GL(NSOpenGLContext)**:iOS 代码本就是 GL(EAGL),`EAGLContext→NSOpenGLContext`、`CAEAGLLayer→CALayer+GL`、skia 用现成 GL 后端 → **到首帧最少改写**。代价:OpenGL 在 macOS 已弃用(但到 macOS 26 仍可用)。
- **Metal(CAMetalLayer)**:`CAEAGLLayer→CAMetalLayer`、需 skia Metal 后端、与 MoltenVK/Vulkan 一致、未来向 → **改写更大**。
- **决定:M1 走 GL(NSOpenGLContext)**——最快出像素;`CADisplayLink→CVDisplayLink` 无论如何必做;Metal 留作后续优化。

### Phase 1 先动哪个
`appframework/graphic_2d/macos/render_context.mm`(GL 上下文,是 EGL/EAGL 错误总源头,解开它 6 个 B 类大概率连带过)。

## Phase 1 结论(已完成核心):macOS 图形后端编译+链接通过 ✅

**产物:`libmacos_graphic.a`(21MB,12 .o,带符号表)** —— 新建 `appframework/graphic_2d/macos/`(15 文件)全部用 Xcode clang 按真实 GN flags 编过并归档成有效 .a。符号验证:render_context.o 含 `NSOpenGLContext/NSOpenGLPixelFormat` + `glGenFramebuffers/glRenderbufferStorage`;vsync.o 含 `CVDisplayLink*`。

**改写(都 mac-gated,留 mac-port 分支):**
- `render_context.mm`(核心重写):`EAGLContext→NSOpenGLContext`(Core Profile 3.2);iOS 的 `renderbufferStorage:fromDrawable:`(CAEAGLLayer 当 GL 后备)桌面无 → 改**离屏 FBO + glRenderbufferStorage**,尺寸取 CALayer bounds×scale;`presentRenderbuffer→glFlush`,**出图责任移交窗口层**。
- `rs_vsync_client_ios.mm`(重写):`CADisplayLink→CVDisplayLink`(MacVsyncDriver,**回调在独立高优先级线程**,窗口层接入要注意)。
- `rs_surface_gpu.mm`/`rs_system_properties.cpp`:CAEAGLLayer→void*(CALayer)、CheckVulkanEnable 桩。
- 直抄(纯 C++):rs_accessibility/ashmem/frame_report/marshalling/render_pipeline_client 等。
- 共享层:`render_context.h/render_context_gl.h` 的 `ROSEN_IOS`→`ROSEN_IOS||ROSEN_MAC` 复用 EGL-type shim → **Phase 0 那 6 个 EGL/egl.h not found 从根上解决**(桌面走 NSOpenGLContext,不碰真 EGL)。
- `appframework/graphic_2d/BUILD.gn`:mac 从 `ios:ios` 改指 `macos:macos`(不污染 iOS)。

**两个硬坎(已解):**
1. **预编译 OHOS clang-15 编不了 macOS 26.4 SDK 的 ObjC 头**(`NSUInteger unknown`,裸 `#import <AppKit/AppKit.h>` 都炸)→ **mac 工具链改用 Xcode clang**(`use_xcode_clang=true`,绝对路径),并门掉 OHOS-only flag。这正是 iOS 工具链用 Xcode clang 的原因。
2. **NSOpenGLContext→CALayer 出图成不成立?——成立**,但形态与 iOS 不同:iOS layer 直接当 GL 后备;桌面 GL 没这能力 → 用**离屏 FBO 渲染**,出图移交窗口层。推荐窗口层走 **CAOpenGLLayer/IOSurface 零拷贝**贴 FBO 纹理(备选:FBO 读回 CPU buffer 贴 CALayer.contents)。render_context.mm 已实现到 FBO+glFlush,出图接口留好。

**未完全通(如实)**:全 GN `ninja` 出 .a 被一个 third_party host 工具挡住——`image_native`→`third_party/ffmpeg` 的 BUILD.gn(只读)给 mac 选了 `ohos_config.sh`,里面用 **GNU `sed -i`**(BSD/mac sed 报错;ffmpeg 的 `android_mac_config.sh` 才是 BSD sed)。因 ffmpeg 只读,没法直接 mac-gate。**与 GL 后端无关**——所以用真实 GN flags 单独编 .o 归档成 .a 证明后端 compile+link 成立。

**最终 gn args**:`target_os=mac target_cpu=arm64 mac_use_ios_backend=true graphic_2d_feature_enable_vulkan=false skia_enable_fontmgr_ohos=false skia_use_fonthost_mac=false use_xcode_clang=true`。
